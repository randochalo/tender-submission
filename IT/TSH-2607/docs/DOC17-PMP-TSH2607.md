## 3. MANAGEMENT PROCESSES

### 3.1 Planning Process

| Activity | Frequency | Output |
|----------|-----------|--------|
| Sprint Planning | Bi-weekly | Sprint backlog |
| Release Planning | Monthly | Release plan |
| Resource Planning | Monthly | Resource forecast |
| Budget Review | Monthly | Budget report |

### 3.2 Monitoring Process

| Metric | Target | Review |
|--------|--------|--------|
| Schedule variance | < 10% | Weekly |
| Cost variance | < 10% | Monthly |
| Defect density | < 5/FP | Per sprint |
| Test coverage | > 80% | Per sprint |

## 4. COMMUNICATION MANAGEMENT

### 4.1 Communication Matrix

| Stakeholder | Information | Frequency | Method |
|-------------|-------------|-----------|--------|
| Project Board | Strategic status | Monthly | Meeting + Report |
| MCMC PMO | Detailed status | Weekly | Email + Dashboard |
| End Users | Updates, training | As needed | Portal + Email |
| Dev Team | Technical updates | Daily | Stand-up |

### 4.2 Reporting Schedule

| Report | Audience | Frequency | Due Date |
|--------|----------|-----------|----------|
| Project Status Report | Board | Monthly | 5th of month |
| Sprint Report | PMO | Bi-weekly | After sprint |
| Risk Register | All | Weekly | Every Monday |
| Budget Report | Board | Monthly | 10th of month |

## 5. RISK MANAGEMENT

### 5.1 Risk Process

1. **Identification:** Weekly risk review
2. **Assessment:** Probability/Impact matrix
3. **Planning:** Mitigation strategies
4. **Monitoring:** Risk register updates

### 5.2 Risk Categories

| Category | Examples | Owner |
|----------|----------|-------|
| Technical | Integration issues | Tech Lead |
| Schedule | Resource delays | PM |
| Financial | Budget overrun | PM |
| External | Regulatory changes | BA |

## 6. QUALITY MANAGEMENT

### 6.1 Quality Standards

| Aspect | Standard | Verification |
|--------|----------|--------------|
| Code quality | ISO 25010 | Code review |
| Documentation | IEEE 830 | Peer review |
| Testing | ISTQB | Test reports |
| Security | KRISA | VAPT |

### 6.2 Quality Gates

| Gate | Criteria | Approver |
|------|----------|----------|
| Requirements | Approved SRS | BA |
| Design | Approved SDS | Architect |
| Code Complete | 80% coverage | Tech Lead |
| UAT Ready | Zero critical | QA |
| Production | Security pass | Security Lead |

---

## 7. CHANGE MANAGEMENT

### 7.1 Change Control Process

1. **Request:** Submit change request form
2. **Impact:** Assess schedule/cost/quality
3. **Approval:** CCB decision
4. **Implementation:** Execute approved changes
5. **Review:** Post-implementation review

### 7.2 Change Authority

| Change Type | Authority | Timeline |
|-------------|-----------|----------|
| Minor (< RM 10K) | PM | 2 days |
| Medium (RM 10K-50K) | PM + Sponsor | 1 week |
| Major (> RM 50K) | Project Board | 2 weeks |

## 8. ISSUE MANAGEMENT

| Severity | Response | Escalation |
|----------|----------|------------|

| High | 1 day | Sponsor |
| Medium | 3 days | PM |
